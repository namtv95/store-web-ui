# Expanded Vendor Inventory

React + Vite shop browser for the Crimson Desert mod:
https://www.nexusmods.com/crimsondesert/mods/2281

Browse 2,000+ shop items with icons, search by name/key/shop, and view dye color palettes.

## Development

Requirements: Node.js 20+

```bash
npm install
npm run dev
```

Open http://localhost:5173

## Build

```bash
npm run build
npm run preview
```

For GitHub Pages project site (`namtv95.github.io/crimson-dersert/`):

```bash
VITE_BASE_PATH=/crimson-dersert/ npm run build
```

## GitHub Pages Deploy

This repo uses GitHub Actions (`.github/workflows/deploy.yml`).

**One-time setup (any branch):**

1. Open repository **Settings** → **Pages**
2. Set **Build and deployment** → **Source** = **GitHub Actions**

**Deploy from `store-browser-web` (non-default branch):**

Option A — push (auto deploy):

```bash
git push origin store-browser-web
```

The workflow runs on push to `main` or `store-browser-web`.

Option B — manual deploy (no push needed):

1. Open **Actions** → **Deploy to GitHub Pages**
2. Click **Run workflow**
3. Choose branch: `store-browser-web`
4. Click **Run workflow**

**Live URL:** `https://namtv95.github.io/crimson-dersert/`

> GitHub Pages always serves from the **latest successful deploy**, no matter which branch built it. You do **not** need to change the default branch.

### Fix: environment protection rules

If deploy fails with:

`Branch "store-browser-web" is not allowed to deploy to github-pages due to environment protection rules`

the **build succeeded** but GitHub blocked the deploy step. Fix it in repo settings:

1. Open **Settings** → **Environments** → **github-pages**
2. Under **Deployment branches**, choose one:
   - **All branches** (easiest), or
   - **Selected branches** → add `store-browser-web`
3. If **Required reviewers** is enabled, either:
   - approve the pending deployment in **Actions** → workflow run → **Review deployments**, or
   - turn off required reviewers for `github-pages`
4. Re-run the failed workflow: **Actions** → failed run → **Re-run all jobs**

Alternative: merge `store-browser-web` into `main` and deploy from `main` (if `main` is already allowed).

## Project structure

```
assets/                # Item/vendor/dye images (used by web + Store Editor)
  icons/               # {key}.webp, {key}_9.webp for Disc9
  vendors/             # Shop NPC portraits
  dye/                 # Dye tab icons
public/
  store_info.json      # Shop inventory data
  dye_color.json       # Dye color groups
  assets -> ../assets  # Symlink for Vite static copy
src/
  components/          # React UI
  hooks/               # Data loading
  utils/               # Search, sort, icon paths
```

## Data files

- `store_info.json` — generated by parent repo `parser/4_get_store_info.py`
- `dye_color.json` — generated by parent repo `7_convert_dye_raw_colors.py`

## Legacy files

The old vanilla `app.js` and `styles.css` are replaced by this Vite + React app.
Static assets and JSON remain in `public/`.
